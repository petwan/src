你是一个资深的数据工程师，帮我解决一下自动驾驶数据采集后的信号分析与场景提取，我有多个摄像头（30fps）和多个激光雷达（10hz）数据，同时我还记录了所有的车辆总线数据（有的数据是100Hz，有的数据是50hz），我需要进行以下检查：
1. 所有传感器数据的时间戳要都是递增的，同时，选择一个主传感器，要求所有传感器的时间戳都是对齐的（即所有主传感器的采样正常，且主传感器的每一帧下都有其他所有传感器的数据，在一定时间差之内），并使用mask标记对应的数据
2. 筛选出AEB激活的场景（因为数据的稀疏性，AEB的那一列数据并不连续，中间某些行可能有其他的数据产生导致数据为null，我需要能够将AEB的信号做连续处理，不希望中间出现null的情况），当然检测后的信号还是放在mask列便于后续的filter处理
3. 部分信号可能需要进行diff处理，null的数据可能会影响diff的处理，我需要你帮我解决一下。
4. 最终，多个条件的叠加后，形成一个符合条件的mask列，然后进行时间范围确定的时候，需要能够支持 max_allowed_gap 允许最大的gap时间间隔等条件，便于将两段事件合并等操作，并将合并的时间范围写入到一个新的文件中，便于后续的场景检索。
帮我设计一下合理的架构，并给出这么设计的原因。
